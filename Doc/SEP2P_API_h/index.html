SEP2P_API_API INT32 SEP2P_SendMsg(
    const CHAR *pDID,
    INT32 nMsgType,
    const CHAR *pMsgData,
    INT32 nMsgDataSize);
Parameters
pDID

type: const CHAR*; means: the device's did..

nMsgType

INT32 nMsgType command type, you can refer to SEP2P_ENUM_MSGTYPE enum.

pMsgData

CHAR* pIOData command data.

nMsgDataSize

INT32 nIODataSize the size of data.

Return Value

INT32 Returns ERR_SEP2P_NOT_INITIALIZED, ERR_SEP2P_INVALID_PARAMETER, ERR_SEP2P_NO_CONNECT_THIS_DID, ERR_SEP2P_INVALID_MSG_TYPE,ERR_SEP2P_NO_SUPPORT_THIS_CODECID or ERR_SEP2P_SUCCESSFUL

Discussion

Send the command that control the device or set the device's params, or get the device's params..


For example:
(1)start video: INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_START_VIDEO, NULL, 0);
(2)stop video: INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_STOP_VIDEO, NULL, 0);
(3) start audio: INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_START_AUDIO, NULL, 0);
(4) stop audio: INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_STOP_AUDIO, NULL, 0);
(5) start talk:INT32 nRet=SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_START_TALK, NULL, 0);
(6) stop talk: INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_STOP_TALK, NULL, 0);
(7) control the device's ptz:
{
MSG_PTZ_CONTROL_REQ ptzMsg;
ptzMsg.nCtrlCmd = PTZ_CTRL_LEFT;
ptzMsg.nCtrlParam = 0;
const CHAR* pMsgData = (CHAR*)&ptzMsg;
INT32 nMsgDataSize = (INT32)sizeof(MSG_PTZ_CONTROL_REQ);
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_PTZ_CONTROL_REQ, pMsgData, nMsgDataSize);
}
(8) // send the get camera's params command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_CAMERA_PARAM_REQ, NULL, 0);
(9)// send the get camera's wifi command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_CURRENT_WIFI_REQ, NULL, 0);
(10)// send the get camera's wifi list command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_WIFI_LIST_REQ, NULL, 0);
(11) // send the get camera's user command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_USER_INFO_REQ, NULL, 0);
(12)// send the get camera's datetime command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_DATETIME_REQ, NULL, 0);
(13)// send the get camera's ftp command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_FTP_INFO_REQ, NULL, 0);
(14)// send the get camera's sdcard schedule command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_SDCARD_REC_PARAM_REQ, NULL, 0);
(15)// send the get camera's hard soft version information command to device
INT32 nRet = SEP2P_SendMsg((CHAR *)[self.m_sDID UTF8String], SEP2P_MSG_GET_DEVICE_VERSION_REQ, NULL, 0);

SEP2P_SendTalkData

SEP2P_SendTalkData(const CHAR* pDID, CHAR* pData, INT32 nDataSize, UINT64 nTimestamp). 

//1>nDataSize>1024 bytes                -- ERR_SEP2P_WRITTEN_SIZE_TOO_BIG 
//2>(nDataSize%40)!=0 for G726        -- ERR_SEP2P_INVALID_PARAMETER 
//3>nDataSize<0                                        -- ERR_SEP2P_INVALID_PARAMETER 
//4>nDataSize!=256 for ADPCM        -- ERR_SEP2P_INVALID_PARAMETER 
SEP2P_API_API INT32 SEP2P_SendTalkData(
    const CHAR *pDID,
    CHAR *pData,
    INT32 nDataSize,
    UINT64 nTimestamp);
Parameters
pDID

type: const CHAR*; means: the device's did..

type: CHAR*; means: the talk data want to be sent..

type: INT32; means: the size of data..

type: UINT64; means: the talk's timestamp..

Return Value

INT32 Returns ERR_SEP2P_NOT_INITIALIZED, ERR_SEP2P_INVALID_PARAMETER, ERR_SEP2P_NO_CONNECT_THIS_DID, ERR_SEP2P_WRITTEN_SIZE_TOO_BIG, ERR_SEP2P_STOPPED_TALK or ERR_SEP2P_SUCCESSFUL.

Discussion

Only the device's status is connected, the interfact is valid. (you can find device's status by SEP2P_MSG_CONNECT_STATUS).Before you send talk audio stream data, you should first send open device's talk channel by send talk cgi command. You can send it by SEP2P_SendMsg() function (the msgtype is SEP2P_MSG_START_TALK), then you can get the result by POnRecvMsgCallback call back funtion (the msgtype is SEP2P_MSG_START_TALK_RESP, you can get a MSG_START_TALK_RESP data, if the result in MSG_START_TALK_RESP equal to 0, it's means that the talk channel be opened succefully, you can send talk data at once; other wise, if the result = 1, means that someone has used the talk channel, you must wait it be free, if the result= 2, it's means that some error happen );

POnStreamCallback

The callback function. The function will be excute when receive video stream, audio stream, you can distinguish stream type by the nStreamType param. you should register the call back function before you open the video stream or open the audio stream. 

typedef INT32 ( *POnStreamCallback)( 
    CHAR *pDID, 
    CHAR *pData, 
    UINT32 nDataSize, 
    VOID *pUserData);
Parameters
pDID

CHAR* pDID:camera's DID.

pData

CHAR* the return data .

nDataSize

INT32 the returned data size.

pUserData

VOID* pUserData the information about caller.

Discussion

The callback function. The function will be excute when receive video stream, audio stream, you can distinguish stream type by the nStreamType param. you should register the call back function before you open the video stream or open the audio stream. if the returned stream is video stream, the data contain a stream head which you can refer to STREAM_HEAD define, the stream head tell you the size of video frame. if the video type is mjpeg, you can direct convert the char* data to image for display, otherwise, if the video type is h264, you should first decode the data, then convert the data to image data. if the stream is audio stream, it also contain a stream head that define in STREAM_HEAD, you can get the size of the audio data and the audio encode type by the STREAM_HEAD. for example,if the audio encode type is adpcm, you should first decode the adpcm data, if the device's type is L series, the audio encode type is adpcm, The M series device audio encode type is g726. The most import thing you should known that the call back can be register successfully after you called the SEP2P_Connect() api .

tag_stStreamHead, STREAM_HEAD define the Stream head packet information, it's used for difference between audio stream and video stream, it's also tell you the stream channel. 

typedef struct tag_stStreamHead { 
    UINT32 nCodecID; // refer to SEP2P_ENUM_AV_CODECID 
    CHAR nParameter; // Video: refer to SEP2P_ENUM_VIDEO_FRAME.   Audio:(samplerate << 2) | (databits << 1) | (channel) 
    CHAR nLivePlayback;// Video: 0:live video or audio;  1:playback video or audio 
    CHAR reserve1[2];  
    UINT32 nStreamDataLen; // Stream data size after following struct 'STREAM_HEAD' 
    UINT32 nTimestamp; // Timestamp of the frame, in milliseconds 
    CHAR reserve2[8]; 
}STREAM_HEAD;
Fields
nCodecID

type: UINT32; means: refer to SEP2P_ENUM_AV_CODECID, tell you the video stream type.

nParameter

type: UINT32; means: Video: refer to SEP2P_ENUM_VIDEO_FRAME. Audio:(samplerate << 2) | (databits << 1) | (channel), if it is video stream, the param means the first frame type. if current stream is audio stream , the param means the audio samplerate, databits ,channel.

nLivePlayback

type: CHAR; means: Video: refer to SEP2P_ENUM_VIDEO_RESO

reserve1

type: CHAR*; means: no used, prepared for future use.

nStreamDataLen

type: CHAR; means: no used, prepared for future use.

nTimestamp

type: CHAR; means: no used, prepared for future use.

reserve2

type: CHAR; means: no used, prepared for future use.

Discussion
